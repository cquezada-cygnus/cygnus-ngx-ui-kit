Inputs

Los campos de entrada permiten a los usuarios introducir texto en una interfaz de usuario. Suelen aparecer en formularios y cuadros de diálogo. Los campos de entrada en móviles deben tener un tamaño de texto de al menos 16px para evitar el zoom automático en los navegadores móviles.

Cómo usar cygnus-input

Importar el componente de la biblioteca ngx-cygnus-ui en el archivo .ts del componente donde lo vas a implementar.

En el documento .html del componente, posicionar la plantilla del input y agregar la configuración según corresponda.
  • [inputCustomType]: Recibe un string que indica el tipo de input que se va a utilizar. Existen los tipos: "base", "fieldset-legend-label", "label-top", "label-interactive", "floating", "file".
  • [textPlaceholder]: Recibe un string que indica el texto del placeholder.
  • [textHint]: Recibe un string que indica el texto que va debajo del input para entregar instrucciones al usuario.
  • [textLabel]: Recibe un string que indica el texto que va arriba del input a modo de encabezado.
  • [inputSize]: Recibe un string que indica el tamaño del input. Si no se indica, tiene un tamaño estándar. Puede tener los siguientes valores: 'sm' y 'lg'.
  • [inputColor]: Recibe un string que indica el color del input. Puede recibir los siguientes valores: 'success', 'warning', 'error'. Si no recibe nada, el color es estándar.
  • [iconAsset]: Recibe un string que indica la url de un ícono dentro de la biblioteca ngx-cygnus-icons.
  • [iconColor]: Recibe un string que indica el color de un ícono dentro del input. Puede recibir los siguientes valores: 'success', 'warning', 'error'. Si no recibe nada, el color es estándar.
  • [iconState]: Recibe un boolean: true para identificar el estilo del input cuando es ícono de success/warning/error. Por defecto su valor es false.
  • [iconSize]: Recibe un string que indica el tamaño del ícono dentro del input. Por defecto tiene el valor '19px'.
  • [iconPosition]: Recibe un string que indica la posición del ícono dentro del input. Su valor puede ser 'right' o 'left'. Por defecto tiene el valor 'right'.
  • [pseudoIconCLPPhone]: Recibe un boolean. Por defecto su valor es false. Se utiliza para indicar el comienzo del número de teléfono para Chile.
  • [hintColor]: Recibe un boolean. Por defecto su valor es false. Se utiliza para indicar si el texto por debajo del input se va a colorear o si va a mantener un color estándar.
  • [inputDisabled]: Recibe un boolean. Por defecto su valor es false. Se utiliza para desabilitar o no el input.
  • [typePassword]: Recibe un boolean. Por defecto su valor es false. Se utiliza para indicar si hay que ocultar el contenido del input en caso de que sea para indicar un password.
  • [inputClearValue]: Recibe un boolean. Por defecto su valor es false. Se utiliza para indicar si se requiere que se limpie el input.
  • [initializeInputValue]: Recibe un string para inicializar el valor del input.
  • [control]: Se utiliza para conectar el input a un formulario.

Cómo usar cygnus-custom-table

Importar el componente de la biblioteca ngx-cygnus-ui y algunas interfaces que podrías necesitar en el archivo .ts del componente donde lo vas a implementar.

Crear la variable que contenga los datos del documento JSON y aquellas que contengan datos que requiera según la configuración deseada.

En el documento .html del componente, posicionar la plantilla de la tabla y agregar la configuración según corresponda.

cygnus-custom-table simple

Tabla recibe cualquier tipo de JSON y lo renderiza en una tabla.

"CLIENTES" corresponde a un JSON con los datos que se van a mostrar. En [dataTable] se deben entregar los datos que se quieren visualizar en la tabla.

'stripped-hover' es un tipo de tabla. Se encuentran implementados los siguientes tipos de tabla: "basic", "stripped", "hover", "stripped-hover", "double" y "select". Stripped-hover se refiere a que habrá una franja de color alternada por cada elemento de la tabla, y además al posicionar el mouse sobre algún elemento, éste se va a destacar.

[maxCounter] se implementa cuando uno quiere agregar paginación a la tabla. Se debe indicar en cuántas páginas se quiere dividir la tabla y por dentro el componente dividirá los elementos entre todas las páginas. Si hay demasiadas páginas, van a quedar algunas páginas vacías.

[showSearch] se utiliza para elegir si se quiere mostrar o no el buscador en la cabecera de la tabla. el buscador busca cualquier string dentro de la tabla y muestra los datos que coinciden automáticamente. Si no hay nada escrito en el buscador, se muestra toda la tabla.

cygnus-custom-table double

Tabla recibe cualquier tipo de JSON y lo renderiza en una tabla. 'double' indica que un casillero de la tabla tiene doble dato.

"CLIENTES" corresponde a un JSON con los datos que se van a mostrar. En [dataTable] se deben entregar los datos que se quieren visualizar en la tabla.

'double' es un tipo de tabla. Se encuentran implementados los siguientes tipos de tabla: "basic", "stripped", "hover", "stripped-hover", "double" y "select". Double se refiere a que habrán elementos en la tabla que se fusionarán en el mismo td. Para saber cuáles elementos se van a fusionar, debes indicar qué keys del JSON se van a fusionar utilizando [doubleKeyUp1] (se posiciona el contenido de la llave arriba a la izquierda), [doubleKeyUp2] (se posiciona el contenido de la llave arriba a la derecha) y [doubleKeydown] (se posiciona el contenido de la llave abajo). Está diseñado para posicionar el nombre, apellido y correo.

[maxCounter] se implementa cuando uno quiere agregar paginación a la tabla. Se debe indicar en cuántas páginas se quiere dividir la tabla y por dentro el componente dividirá los elementos entre todas las páginas. Si hay demasiadas páginas, van a quedar algunas páginas vacías.

[showSearch] se utiliza para elegir si se quiere mostrar o no el buscador en la cabecera de la tabla. el buscador busca cualquier string dentro de la tabla y muestra los datos que coinciden automáticamente. Si no hay nada escrito en el buscador, se muestra toda la tabla.

Si se implementa [maxCounter] (paginación) y [showSearch] (filtro de búsqueda) en la misma tabla, se actualiza la paginación con las busquedas pero van quedando páginas vacías.

cygnus-custom-table can edit

Tabla recibe cualquier tipo de JSON y lo renderiza en una tabla. Al presionar editar, te permite modificar las partes que indiques y envia los cambios para que los puedas guardar.

[tdEditArr] cuando se encuentra implementado se utiliza para indicar qué elementos de la tabla se pueden editar. Debes entregar un arreglo que contenga strings que correspondan a las keys del JSON que quieres que se puedan editar. Al presionar el link "editar" en la última columna, aparece el modo edición del documento para los elementos que se indicó que se pueden editar.

(emitModifiedData) es un evento que entrega el resultado de la edición del documento de la tabla. Contiene el elemento original y el resultado de la edición.

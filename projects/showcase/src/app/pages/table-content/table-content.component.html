Tablas

Elige la mejor tabla para una visualización de datos responsiva y en forma de cuadrícula. Se puede personalizar fácilmente con búsqueda, paginación, botones de acción y más

Cómo usar cygnus-custom-table

Importar el componente de la biblioteca ngx-cygnus-ui y algunas interfaces que podrías necesitar en el archivo .ts del componente donde lo vas a implementar.

Crear la variable que contenga los datos del documento JSON y aquellas que contengan datos que requiera según la configuración deseada.

En el documento .html del componente, posicionar la plantilla de la tabla y agregar la configuración según corresponda.

cygnus-custom-table simple

[dataTable] recibe cualquier tipo de JSON y lo renderiza en una tabla.

"CLIENTES" corresponde a un JSON con los datos que se van a mostrar. En [dataTable] se deben entregar los datos que se quieren visualizar en la tabla.

'stripped-hover' es un tipo de tabla. Se encuentran implementados los siguientes tipos de tabla: "basic", "stripped", "hover", "stripped-hover", "double" y "select". Stripped-hover se refiere a que habrá una franja de color alternada por cada elemento de la tabla, y además al posicionar el mouse sobre algún elemento, éste se va a destacar. En [tableType] se debe entregar el tipo de tabla.

[showSearch] se utiliza para elegir si se quiere mostrar o no el buscador en la cabecera de la tabla. el buscador busca cualquier string dentro de la tabla y muestra los datos que coinciden automáticamente. Si no hay nada escrito en el buscador, se muestra toda la tabla.

[tdFormatKeysArr] Se utiliza cuando uno quiere dar un formato específico a ciertos números que pueden ser rut, dinero o números. Se debe entregar un arreglo de objetos de tipo TdFormat donde cada key corresponde al nombre de la columna donde habrá algún tipo de número y cada format corresponde al tipo de formato que se utilizará.

"CLIENTES" corresponde a un JSON con los datos que se van a mostrar. En [dataTable] se deben entregar los datos que se quieren visualizar en la tabla.

Double se refiere a que habrán elementos en la tabla que se fusionarán en el mismo td. Para saber cuáles elementos se van a fusionar, debes indicar qué keys del JSON se van a fusionar utilizando [doubleKeyUp1] (se posiciona el contenido de la llave arriba a la izquierda), [doubleKeyUp2] (se posiciona el contenido de la llave arriba a la derecha) y [doubleKeydown] (se posiciona el contenido de la llave abajo). Está diseñado para posicionar el nombre, apellido y correo.

cygnus-custom-table can edit

[dataTable] recibe cualquier tipo de JSON y lo renderiza en una tabla. Al presionar editar, te permite modificar las partes que indiques y envia los cambios para que los puedas guardar.

"EMPRESAS" corresponde a un JSON con los datos que se van a mostrar. En [dataTable] se deben entregar los datos que se quieren visualizar en la tabla.

En [tableType] se debe entregar el tipo de tabla.

[tdEditArr] cuando se encuentra implementado se utiliza para indicar qué elementos de la tabla se pueden editar. Debes entregar un arreglo que contenga strings que correspondan a las keys del JSON que quieres que se puedan editar. Al presionar el link "editar" en la última columna, aparece el modo edición del documento para los elementos que se indicó que se pueden editar.

'NOMBRE DE FANTASIA', 'EMAIL' y 'DIRECCION' corresponden a los nombres de las columnas que serán editables.

(emitModifiedData) es un evento que entrega el resultado de la edición del documento de la tabla. Contiene el elemento original y el resultado de la edición.

cygnus-custom-table Filtrar Columnas

A veces una tabla puede tener muchísimas columnas y nos interesa visualizar solo algunas de ellas. Para eso debemos entregarle a la tabla un arreglo de strings donde cada string corresponda a un key de la columna de la tabla que se quiera mostrar. Las demás columnas se ignorarán al renderizar la vista.

"COMPANY_COLAB" corresponde a un JSON con los datos que se van a mostrar. En [dataTable] se deben entregar los datos que se quieren visualizar en la tabla.

'stripped-hover' es un tipo de tabla. Se encuentran implementados los siguientes tipos de tabla: "basic", "stripped", "hover", "stripped-hover", "double" y "select". Stripped-hover se refiere a que habrá una franja de color alternada por cada elemento de la tabla, y además al posicionar el mouse sobre algún elemento, éste se va a destacar. [tdFormatKeysArr] Se utiliza cuando uno quiere dar un formato específico a ciertos números que pueden ser rut, dinero o números. Se debe entregar un arreglo de objetos de tipo TdFormat donde cada key corresponde al nombre de la columna donde habrá algún tipo de número y cada format corresponde al tipo de formato que se utilizará.

[filtroColumnas] se utiliza para indicar qué columnas se van a mostrar para ocultar todas las demás. Si está configurado, aparecerá un ícono de ojo en la tabla, el cual es un toggle que permitirá mostrar todo o mostrar sólo las columnas seleccionadas en filtroColumnas al hacerle click. Debes entregarle un arreglo de strings donde los strings correspondan a las llaves de las columnas que quieras mostrar.

Selectores de Búsqueda

El select de búsqueda aplica a los usuarios a introducir sus consultas, afinar los resultados y mostrar fácilmente la información relevante. La barra de búsqueda es claramente visible colocándola en un lugar común y esperado por lo general, en la parte superior de la página- y haciéndola lo suficientemente grande como para que los usuarios la vean fácilmente. Además, se utiliza el icono de una lupa para representar la función de búsqueda, ya que es el símbolo más reconocido.

Selector de búsqueda con botón

Este ejemplo de una entrada de búsqueda con el botón de búsqueda ubicado dentro del campo de entrada. Este diseño compacto permite a los usuarios escribir su consulta y hacer clic inmediatamente en el botón de búsqueda dentro del mismo elemento, ahorrando espacio.


Importar el componente de la biblioteca ngx-cygnus-ui y algunas interfaces que podrías necesitar en el archivo .ts del componente donde lo vas a implementar.

En el archivo .ts crea un arreglo cuyos elementos cumplan con la interfaz SelectGeneric para desplegarlos en el selector. Para obtener el resultado de la selección, puedes usar un método para capturar el resultado.

En el documento .html del componente, posicionar la plantilla del selector y agregar la configuración según corresponda.

Puedes cambiar el comportamiento del selector con [showOptionsAutomatically] ingresando un boolean. Al estar activado, mostrará las opciones mientras escribes en lugar de mostrar las opciones cuando presionas el botón buscar.

Al indicar [multisearch]="true" Se obtiene un componente de búsqueda de palabras clave que se podrán ir seleccionando y el componente emitirá en un evento los elementos seleccionados. Puedes recibir los elementos seleccionados de (outputMultisearch) y utilizarlos para filtrar los elementos de una tabla por ejemplo. Si se elimina alguno de los elementos seleccionados, se vuelve a emitir el evento actualizado.

[multisearch] funciona mejor cuando [showOptionsAutomatically]="true", por lo tanto aunque no se indique explícitamente, por defecto showOptionsAutomatically es true cuando multisearch es true.

Selector con menú desplegable

Este ejemplo de entrada de búsqueda incluye un menú desplegable para filtrar las categorías de búsqueda. Este diseño ayuda a refinar las búsquedas y brinda resultados más precisos.

Selector con menú desplegable y multisearch

Este componente recibe el listado de categorías y las opciones por categoría como en la sección anterior, pero además permite seleccionar más de una opción dentro de una misma categoría, las cuales se mostrarán debajo del selector y se pueden eliminar con un click cuando no sean necesarias.

Selector con categorias, secciones, busqueda por todas las categorías y búsqueda dentro de la categoría

Este componente recibe un JSON con los cargos. Cada cargo tiene categoría y sección. El mismo componente reconoce las categorías y secciones. Al abrir el desplegable por primera vez, se verán las categorías. Si se selecciona una categoría se verán las secciones y los cargos dentro de cada sección.

Puedes elegir desde el menú el cargo o puedes escribirlo. Si no has seleccionado categoría, te mostrará el cargo de cualquier categoría que coincidad con tu búsqueda. Si seleccionaste categoría, te mostrará el cargo que coincida con tu búsqueda dentro de la categoría seleccionada.

Puedes de-seleccionar la categoría con la opción volver.
